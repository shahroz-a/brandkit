# Style Guide

## Product UI

- Prioritize the usable editor over marketing pages.
- Keep controls compact, predictable, and keyboard accessible.
- Use icons for common tool actions.
- Keep cards for repeated items, modals, and framed tools.
- Avoid blank states; every empty state needs an illustration, headline, description, action, and useful next step.

## Generated Assets

- Preserve safe areas for logos and text.
- Maintain WCAG AA contrast when possible.
- Scale text by content length, not viewport width.
- Support long names, emoji, RTL text, and non-Latin scripts gracefully.

## Code

- Keep core logic pure.
- Use explicit data shapes.
- Prefer small functions with direct names.
- Add comments only when they clarify non-obvious behavior.
